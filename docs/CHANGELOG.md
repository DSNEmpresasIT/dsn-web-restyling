pueda seguir la evolución técnica y de contenido sin depender solo del
historial de Git.

## Criterio de registro

Usa este changelog para cambios que impactan funcionalidad, contenido visible,
decisiones técnicas, o flujo de trabajo del equipo.

### Added
- Nuevas páginas, componentes, rutas, secciones o capacidades.

### Changed
- Cambios de comportamiento, UX, arquitectura o estructura de datos.

### Fixed
- Correcciones de bugs que afectaban el resultado en desarrollo o producción.

### Docs
- Cambios en guías de trabajo, handoff, arquitectura o convenciones.

## Regla rápida de actualización

Actualiza `Unreleased` en cada PR que cambie comportamiento o contexto del
proyecto. Cuando se haga un release, mueve los puntos a una versión con fecha.
